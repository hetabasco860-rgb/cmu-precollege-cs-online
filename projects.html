Overview

The Final Project is intended to demonstrate readiness for university-style computer science work, including the ability to plan a scope, implement correctly, test systematically, and communicate technical decisions clearly.

Submission Policy

Single submission: The Final Project is evaluated from a single official submission. Students are expected to complete planning, implementation, testing, and documentation prior to submission.

Because the submission is official, projects should be complete and internally consistent: the code should match the written description, and usage instructions should match actual behavior.

Project Expectations

The final project is expected to meet the following baseline expectations:

  • Clear goal: A well-defined purpose that can be explained in a few sentences.
  • Functional completeness: The program runs reliably and supports the documented workflow.
  • Appropriate complexity: Sufficient depth to demonstrate programming and problem-solving skills.
  • Readable implementation: Clear structure, consistent naming, and maintainable organization.
  • Testing evidence: Representative test cases and notes describing how correctness was verified.

Required Submission Package

The final submission should include all of the following components:

  • Source code: Organized files with readable structure and consistent style.
  • Project write-up: A concise technical report describing purpose, approach, and key decisions.
  • Usage instructions: How to run the project, required inputs, and expected outputs.
  • Testing note: A brief list of test cases used and what each test verified.
Documentation standard: Reviewers should be able to run the project using only the instructions provided.

Evaluation Criteria

Projects are evaluated holistically using the criteria below. The rubric is intended to reward correctness, clarity, and evidence of thoughtful engineering decisions.

  • Technical Implementation: correctness, reliability, appropriate use of CS concepts
  • Code Quality: structure, clarity, naming, organization, maintainability
  • Problem Solving: scope selection, trade-offs, iterative improvement, reasoning
  • Communication: documentation quality, explanation clarity, usability

Suggested Project Directions

Students may select a project direction aligned with their interests, including:

  • Interactive application: a multi-feature tool with structured menus and input handling
  • Data project: ingest, transform, and analyze a dataset with structured reporting
  • Algorithmic build: implement a defined algorithm and demonstrate evaluation or comparison
  • Simulation: model system behavior over time and present results clearly

Academic Integrity

All submitted work must reflect the student’s independent understanding and implementation. External libraries may be used when appropriate, provided usage is clearly documented and the student can explain how the library is used.
