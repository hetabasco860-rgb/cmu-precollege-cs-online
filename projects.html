Final Project Overview

The Final Project is the culminating assessment of the program. Students design and implement a complete software artifact that demonstrates programming proficiency, computational thinking, and clear technical communication.

The Final Project is evaluated from a single official submission. Students are expected to plan carefully, validate correctness, and ensure documentation is complete prior to submission.

Project Expectations

  • Scope: A complete, functional program with a clearly defined purpose.
  • Design: Coherent architecture and readable, well-structured code.
  • Correctness: The project behaves as documented across representative test cases.
  • Documentation: Clear usage instructions and a brief technical explanation.

Submission Package

The final submission should include:

  • Source code (organized and readable)
  • A short project write-up describing the goal, approach, and key design decisions
  • Usage instructions (how to run, required inputs, expected outputs)
  • A testing note (examples of test cases used to validate correctness)

Evaluation Criteria

Projects are evaluated holistically using the following criteria:

  • Technical Implementation: correctness, reliability, and appropriate use of CS concepts
  • Code Quality: structure, clarity, naming, and maintainability
  • Problem Solving: evidence of thoughtful design choices and iteration
  • Communication: documentation quality and clarity of explanation

Sample Project Directions

Students may choose among a variety of directions, including:

  • Interactive Application: a menu-driven tool or command-line application with multiple features
  • Data Project: a program that analyzes a dataset and reports structured findings
  • Algorithmic Build: an implementation of a well-defined algorithm with evaluation or benchmarking
  • Simulation: a small-scale simulation demonstrating system behavior over time

Academic Integrity

All submitted work must reflect the student’s own understanding and implementation. External libraries may be used when appropriate, provided usage is clearly documented.
